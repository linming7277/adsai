## 🏗️ 架构设计

### 当前架构

```
Frontend (Makerkit + Next.js)
    ↓ (JWT Token)
Go 微服务 (Cloud Run)
    ├── adscenter ⚠️ 内存缓存、全局状态
    ├── offer (DDD/CQRS)
    ├── billing (Token 管理)
    ├── browser-exec
    ├── siterank
    └── recommendations
    ↓
基础设施 (GCP)
    ├── Supabase PostgreSQL
    ├── Cloud SQL PostgreSQL
    ├── Redis
    └── Pub/Sub
```

### 目标架构

```
Frontend (Makerkit + Next.js) - 保持不变
    ↓ (JWT Token)
Go 微服务 (Cloud Run) - 改进
    ├── adscenter ✅ 依赖注入 + Redis + 断路器 + 测试 + 文档
    ├── offer ✅ 完善事件溯源 + 测试 + 文档
    ├── billing ✅ 完善测试 + 文档
    ├── browser-exec ✅ 测试 + 文档
    ├── siterank ✅ 测试 + 文档
    └── recommendations ✅ 测试 + 文档
    ↓
基础设施 (GCP) + 监控
    ├── Supabase PostgreSQL + 性能优化
    ├── Cloud SQL PostgreSQL + 索引优化
    ├── Redis + 多级缓存
    ├── Prometheus (指标收集)
    └── Grafana (可视化 + 告警)
```

## 🚀 执行指南

### 如何开始执行任务

1. **打开任务文件**:
   ```
   .kiro/specs/architecture-improvement-phase1-3/tasks.md
   ```

2. **选择任务**: 从阶段 1 的第一个任务开始

3. **点击 "Start task"**: 在任务旁边点击开始按钮

4. **执行任务**: Kiro 会根据任务描述执行实现

5. **审查结果**: 任务完成后审查代码变更

6. **继续下一个**: 满意后继续下一个任务

### 执行原则

1. **一次一个任务**: 完成一个任务后停止，等待审查
2. **测试先行**: 编写代码前先编写测试
3. **KISS 原则**: 保持实现简单
4. **向后兼容**: 不破坏现有功能
5. **基于实际架构**: 遵循 MustKnowV6.md 中的架构设计
