#### `/api/v1/identity/register` - 用户注册同步

```go
func (h *Handler) registerUser(w http.ResponseWriter, r *http.Request) {
    firebaseUID := r.Context().Value(auth.UserIDContextKey).(string)
    firebaseUser, _ := h.AuthClient.GetUser(context.Background(), firebaseUID)

    // 检查是否已存在
    var existingUserID string
    err := h.DB.QueryRow(context.Background(), `SELECT id FROM "User" WHERE id = $1`, firebaseUID).Scan(&existingUserID)
    if err == nil {
        // 用户已存在
        return
    }

    // 发布 UserRegisteredEvent
    event := domain.UserRegisteredEvent{
        UserID:       firebaseUID,
        Email:        firebaseUser.Email,
        DisplayName:  firebaseUser.DisplayName,
        Role:         "USER",
    }
    h.EventPublisher.Publish(context.Background(), event)
}
```

**职责**：
- 接收 Firebase 已认证用户
- 将用户信息同步到 PostgreSQL `User` 表
- 发布 `UserRegisteredEvent` 到 Pub/Sub

### 3.3 功能可替代性分析

| Identity 功能 | 当前实现 | 替代方案 | 优势 |
|--------------|---------|---------|------|

### 3.4 结论

**❌ 建议下线**

**理由**：
1. **前端未使用**：0 次 API 调用
2. **功能已被覆盖**：
   - Firebase Authentication 提供用户管理
   - Firestore 存储用户配置
   - Firebase Custom Claims 提供权限管理
3. **数据同步可替代**：使用 Cloud Functions 更优雅
4. **维护成本高**：独立服务、数据库迁移、事件发布

**替代方案**：

#### 方案 1：Cloud Functions for Firebase（推荐）

```typescript
// functions/src/sync-user-to-postgres.ts
import * as functions from 'firebase-functions';
import { Client } from 'pg';

export const syncUserToPostgres = functions.firestore
  .document('users/{userId}')
  .onCreate(async (snap, context) => {
    const userData = snap.data();
    const userId = context.params.userId;

    const client = new Client({
      connectionString: process.env.DATABASE_URL,
    });

    await client.connect();

    try {
      await client.query(`
        INSERT INTO "User" (id, email, display_name, role, created_at)
        VALUES ($1, $2, $3, 'USER', NOW())
        ON CONFLICT (id) DO UPDATE SET
          email = EXCLUDED.email,
          display_name = EXCLUDED.display_name
      `, [userId, userData.email, userData.displayName]);

      console.log(`✅ Synced user ${userId} to PostgreSQL`);
    } finally {
      await client.end();
    }
  });
```

**优势**：
- ✅ 事件驱动，自动触发
- ✅ 无需独立服务，降低维护成本
- ✅ 延迟低（< 1 秒）
- ✅ 自动重试（Cloud Functions 特性）

#### 方案 2：Firebase Custom Claims（权限管理）

```typescript
// scripts/admin/grant-admin-role.ts
import admin from 'firebase-admin';

admin.initializeApp({
  credential: admin.credential.cert(require('../secrets/firebase-adminsdk.json')),
});

async function grantAdminRole(email: string) {
  const user = await admin.auth().getUserByEmail(email);

  // 设置 Custom Claims
  await admin.auth().setCustomUserClaims(user.uid, {
    role: 'admin',
    permissions: ['users:read', 'users:write', 'organizations:manage']
  });

  console.log(`✅ Granted admin role to ${email}`);
}

// 执行
grantAdminRole('admin@autoads.dev');
```

func AdminOnly(next http.Handler) http.Handler {

### 4.3 问题分析

**问题 1：功能重复**
- GoFly Admin 的用户管理 ≈ Makerkit Admin 的用户管理
- 用户需要在两个系统间切换，体验割裂

**问题 2：UI 不一致**
- GoFly Admin 使用 Ant Design Pro（蓝色主题）
- Makerkit 使用 Tailwind CSS + Radix UI（可定制）
- 品牌形象不统一

**问题 3：维护成本**
- 需要维护两套前端代码库
- CI/CD 流水线复杂（两个 Cloud Build 配置）
- 部署资源浪费（两个 Cloud Run 实例）

**问题 4：数据源混乱**
- GoFly Admin 直接调用 Console 服务 API（HTTP 代理）
- Makerkit Admin 查询 Firestore
- 数据一致性问题

### 4.4 结论

**❌ 建议下线 GoFly Admin**

**理由**：
1. **功能重复**：与 Makerkit Admin 70% 重叠
2. **体验分裂**：用户需要在两个系统间切换
3. **维护成本高**：两套前端、两个部署流程
4. **Makerkit 优势明显**：
   - 完整的组织管理（GoFly 缺失）
   - Stripe 深度集成（GoFly 仅简单查看）
   - 专业 SaaS UI/UX（GoFly 为运维风格）

- `/readyz` - 依赖就绪检查（数据库 Ping）

**保留理由**：
- ✅ **无法用 Makerkit 替代**：
  - Console 服务拥有 **2223 行代码**，52 个复杂端点
  - 涉及 PostgreSQL、Firestore、Prometheus、Secret Manager、Pub/Sub 等多数据源
  - Go 的并发能力（goroutines）用于并发查询多个服务
  - 性能关键（SLO 监控需要 < 500ms 响应）

- ✅ **职责独立**：
  - Makerkit Admin：通用 SaaS 管理（用户、组织、订阅）
  - Console 服务：业务专属运维（指标、事件、配置、工具）

- ✅ **架构优势**：
  - 独立扩展（可横向扩容，独立于前端）
  - 故障隔离（Console 挂了不影响用户前端）
  - API 复用（可被其他系统调用，如监控告警）

**为什么不合并到 Makerkit**：

| 维度 | Console 服务（Go） | Makerkit API Routes（Next.js） | 结论 |
|------|-------------------|------------------------------|------|
| **性能** | Go goroutines 并发查询多服务 | Node.js 单线程，性能不足 | ❌ 不适合 |
| **数据库** | pgxpool 连接池，高频查询 | Prisma/ORM，性能较差 | ❌ 不适合 |
| **复杂度** | 2223 行代码，52 个端点 | 需要完全重写 | ❌ 工作量巨大 |
| **维护性** | Go 静态类型，易于重构 | TypeScript 类型推断，容易出错 | ❌ 维护成本高 |
| **扩展性** | 独立横向扩容 | 与前端耦合，无法独立扩展 | ❌ 不适合 |

// functions/src/sync-user-to-postgres.ts
import * as functions from 'firebase-functions';
import { Client } from 'pg';
import { SecretManagerServiceClient } from '@google-cloud/secret-manager';

const secretClient = new SecretManagerServiceClient();

async function getDatabaseURL(): Promise<string> {
  const [version] = await secretClient.accessSecretVersion({
    name: 'projects/gen-lang-client-0944935873/secrets/DATABASE_URL/versions/latest',
  });
  return version.payload.data.toString();
}

export const syncUserToPostgres = functions.firestore
  .document('users/{userId}')
  .onCreate(async (snap, context) => {
    const userData = snap.data();
    const userId = context.params.userId;

    const databaseURL = await getDatabaseURL();
    const client = new Client({ connectionString: databaseURL });

    await client.connect();

    try {
      await client.query(`
        INSERT INTO "User" (id, email, display_name, role, created_at)
        VALUES ($1, $2, $3, 'USER', NOW())
        ON CONFLICT (id) DO UPDATE SET
          email = EXCLUDED.email,
          display_name = EXCLUDED.display_name,
          updated_at = NOW()
      `, [userId, userData.email, userData.displayName || '']);

      functions.logger.info(`✅ Synced user ${userId} to PostgreSQL`);
    } catch (error) {
      functions.logger.error(`❌ Failed to sync user ${userId}:`, error);
      throw error;  // Cloud Functions 会自动重试
    } finally {
      await client.end();
    }
  });

// pkg/middleware/admin.go
func AdminOnly(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // 从 Firebase ID Token 获取 Custom Claims
        tokenString := strings.TrimPrefix(r.Header.Get("Authorization"), "Bearer ")

        token, err := firebaseClient.VerifyIDToken(r.Context(), tokenString)
        if err != nil {
            http.Error(w, "Unauthorized", http.StatusUnauthorized)
            return
        }

        // 检查 role claim
        role, ok := token.Claims["role"].(string)
        if !ok || role != "admin" {
            http.Error(w, "Forbidden: Admin role required", http.StatusForbidden)
            return
        }

        // 将 token 存入 context 供后续使用
        ctx := context.WithValue(r.Context(), "firebaseToken", token)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
