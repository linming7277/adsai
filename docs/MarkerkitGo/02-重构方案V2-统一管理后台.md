### 🔍 深度代码审查发现的问题

#### 1. Console服务职责过重（52个端点）
**问题**：
- Console服务承担了过多职责：SaaS管理 + 业务运营 + DevOps运维
- 52个API端点，功能重叠混乱
- 包含Event Sourcing、ROI分析、死信队列、告警管理等复杂功能
- 与Makerkit Admin功能有5%重叠（用户列表查询）

**影响**：
- 维护成本高（2223行代码，15个handler文件）
- 部署体积大（~150MB Docker镜像）
- 职责不清晰，难以扩展

#### 2. 双管理后台维护负担
**问题**：
- Makerkit Admin（SaaS管理）+ Console静态UI（运营管理）
- 两套UI代码需要同步维护
- 用户体验不一致

**用户反馈**：
> "维护两套管理后台很繁琐，有没有统一的管理后台解决方案？"

## 🎯 V2方案核心目标

### 目标1：Console服务精简
- **从52个端点精简到22个核心端点**（-57%）
- **删除复杂运营功能**（Event Sourcing、ROI、DLQ、告警）
- **聚焦核心管理功能**（用户、Token、套餐、API密钥、配置）

### 目标2：统一管理后台
- **单一UI界面**（仅Makerkit，无Console静态UI）
- **Console变为纯API服务**（无UI状态管理）
- **前后端清晰分离**（Makerkit = Presentation，Console = Data API）

### 目标3：功能聚焦
- **保留核心管理功能**：
  - 用户管理
  - Token管理（包括消耗规则）
  - 套餐管理
  - API密钥管理
  - 配置热更新（限流、限速、业务规则）
- **删除运营监控功能**，使用第三方工具替代

### 架构优势

#### 1. 单一管理后台体验
- ✅ **管理员只需访问一个URL**: `https://www.autoads.dev/admin`
- ✅ **统一的UI/UX**: 基于Makerkit设计系统
- ✅ **无需切换界面**: SaaS管理和业务管理在同一导航下

#### 2. 前后端清晰分离
- ✅ **Makerkit = Presentation Layer**: 负责UI渲染、路由、状态管理
- ✅ **Console = Data API Layer**: 负责PostgreSQL查询、业务逻辑封装
- ✅ **无UI状态管理**: Console只提供RESTful API，无静态文件服务

#### 3. 维护成本显著降低
- ✅ **Console代码精简57%**: 从2223行降至~1200行
- ✅ **API端点减少57%**: 从52个降至22个
- ✅ **Docker镜像缩小47%**: 从~150MB降至~80MB
- ✅ **只维护一套UI代码**: Makerkit React/TypeScript组件

#### 4. 职责聚焦
- ✅ **Console专注核心管理**: 用户、Token、套餐、API密钥、配置
- ✅ **删除DevOps功能**: Event Sourcing、DLQ、告警（使用第三方工具）
- ✅ **删除复杂运营功能**: ROI分析（使用Google Analytics + Stripe Dashboard）

## 📦 Console 服务精简方案

### 保留的核心功能（22个端点）

#### 1. 基础健康检查（4个端点）
```go
✅ GET /healthz                    // Kubernetes健康探针
✅ GET /health                     // 健康检查
✅ GET /readyz                     // 就绪检查（含数据库连接测试）
✅ GET /api/health                 // 健康聚合（Gateway使用）
```

#### 2. 用户管理（2个端点）
```go
✅ GET  /api/v1/console/users              // 用户列表（分页、搜索）
✅ *    /api/v1/console/users/{id}/*       // 用户树形路由
    GET  /api/v1/console/users/{id}        // 用户详情
    GET  /api/v1/console/users/{id}/tokens // 用户Token余额
    PUT  /api/v1/console/users/{id}/role   // 更新用户角色
```

**数据源**: PostgreSQL `users` 表

#### 3. Token管理（6个端点，含新增4个）
```go
✅ GET  /api/v1/console/tokens/stats       // Token统计（总余额、消耗、平均值）

// 🆕 Token消耗规则管理（新增）
✅ GET    /api/v1/console/tokens/rules                 // 规则列表
✅ POST   /api/v1/console/tokens/rules                 // 创建规则
✅ PUT    /api/v1/console/tokens/rules/{id}           // 更新规则
✅ DELETE /api/v1/console/tokens/rules/{id}           // 删除规则
```

**数据源**:
- PostgreSQL `token_balances` 表
- PostgreSQL `token_consumption_rules` 表（新增）

**Token消耗规则数据结构**:
```go
type TokenRule struct {
    ID           string    `json:"id"`
    ServiceName  string    `json:"serviceName"`  // adscenter, batchopen, siterank
    ActionType   string    `json:"actionType"`   // ad_query, batch_open, rank_check
    CostPerUnit  int       `json:"costPerUnit"`  // Token消耗量
    Description  string    `json:"description"`
    CreatedAt    time.Time `json:"createdAt"`
    UpdatedAt    time.Time `json:"updatedAt"`
}
```

#### 4. 套餐管理（通过配置管理实现）
```go
✅ GET  /api/v1/console/config?key=plans.*  // 获取套餐配置
✅ PUT  /api/v1/console/config              // 更新套餐配置
```

**套餐配置示例**（存储在 `console_config` 表）:
```json
{
  "plans.free.tokenGrant": 1000,
  "plans.free.features": ["basic_ads", "manual_batch"],
  "plans.pro.tokenGrant": 10000,
  "plans.pro.priceMonthly": 29900,
  "plans.pro.features": ["all", "auto_batch", "advanced_analytics"],
  "plans.enterprise.tokenGrant": 100000,
  "plans.enterprise.features": ["all", "priority_support", "dedicated_account_manager"]
}
```

#### 5. API密钥管理（4个端点）
```go
✅ GET    /api/v1/console/apikeys              // API密钥列表（隐藏密钥明文）
✅ POST   /api/v1/console/apikeys              // 创建API密钥
✅ *      /api/v1/console/apikeys/{id}         // 密钥详情/更新/删除
✅ POST   /api/v1/console/apikeys/validate     // 密钥验证（内部服务调用）
```

**数据源**: PostgreSQL `api_keys` 表

#### 6. 配置热更新（4个端点）
```go
✅ GET  /api/v1/console/config                    // 配置列表
✅ GET  /api/v1/console/config/history           // 配置历史（审计用）
✅ *    /api/v1/console/config/{key}             // 配置CRUD（支持嵌套key）
✅ GET  /ops/console/config/v1                   // 配置快照（供其他服务读取）
```

**支持的配置项**:
```

rateLimit.batchopen.maxConcurrent         // Batchopen限流：最大并发数
rateLimit.siterank.qpm                    // Siterank限流：查询/分钟
businessRules.adscenter.maxAdsPerRequest  // 业务规则：单次最多查询广告数
businessRules.batchopen.maxUrlsPerBatch   // 业务规则：单批次最多URL数
plans.*                                   // 套餐配置
tokens.rules.*                            // Token消耗规则（配置形式）
```

**数据源**: PostgreSQL `console_config` 表（key-value结构，支持JSON value）

#### 7. Dashboard统计（2个端点）

              <th>服务</th>
              <th>操作类型</th>
              <th>消耗量</th>
              <th>描述</th>
              <th>操作</th>
            </tr>
          </thead>
          <tbody>
            {rules.map((rule) => (
              <tr key={rule.id}>
                <td>{rule.serviceName}</td>
                <td>{rule.actionType}</td>
                <td className="font-mono">{rule.costPerUnit} Token</td>
                <td>{rule.description}</td>
                <td>
                  <Button size="sm" variant="outline" onClick={() => handleEdit(rule)}>
                    编辑
                  </Button>
                </td>
              </tr>
            ))}
          </tbody>
        </table>
      </div>

      {showCreateModal && (
        <CreateRuleModal
          onClose={() => setShowCreateModal(false)}
          onSuccess={fetchRules}
        />
      )}
    </div>
  );
}

#### 2. 配置热更新页面示例

**`apps/frontend/src/pages/admin/config/index.tsx`**

```typescript
import { useState, useEffect } from 'react';
import { consoleApi } from '~/lib/console-api-client';
import Heading from '~/core/ui/Heading';
import TextField from '~/core/ui/TextField';

interface Config {
  key: string;
  value: any;
  description?: string;
  updatedAt: string;
}

export default function ConfigPage() {
  const [configs, setConfigs] = useState<Config[]>([]);
  const [searchTerm, setSearchTerm] = useState('');

  useEffect(() => {
    fetchConfigs();
  }, []);

  async function fetchConfigs() {
    const data = await consoleApi.config.list();
    setConfigs(data);
  }

  const filteredConfigs = configs.filter((c) =>
    c.key.toLowerCase().includes(searchTerm.toLowerCase())
  );

  return (
    <div className="space-y-6">
      <div className="flex items-center justify-between">
        <Heading type={2}>系统配置</Heading>
        <TextField
          placeholder="搜索配置项..."
          value={searchTerm}
          onChange={(e) => setSearchTerm(e.target.value)}
        />
      </div>

      <div className="grid grid-cols-1 gap-4 md:grid-cols-2">
        {filteredConfigs.map((config) => (
          <ConfigCard key={config.key} config={config} onUpdate={fetchConfigs} />
        ))}
      </div>
    </div>
  );
}

function ConfigCard({ config, onUpdate }: { config: Config; onUpdate: () => void }) {
  return (
    <div className="rounded-lg border border-gray-200 bg-white p-6 hover:shadow-md">
      <div className="mb-2 font-mono text-sm text-blue-600">{config.key}</div>
      <div className="mb-4 text-2xl font-bold">
        {JSON.stringify(config.value)}
      </div>
      {config.description && (
        <div className="mb-4 text-sm text-gray-500">{config.description}</div>
      )}
      <div className="flex items-center justify-between">
        <span className="text-xs text-gray-400">
          更新于 {new Date(config.updatedAt).toLocaleString()}
        </span>
        <Button size="sm" variant="outline" onClick={() => handleEdit(config)}>
          编辑
        </Button>
      </div>
    </div>
  );
}
```
