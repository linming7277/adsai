## 🔍 问题诊断

### 原始问题

**现象**: 预发环境访问超时，应用无法响应请求

**错误日志**:
```
Error loading i18n file: ~/locales/zh-CN/common.json: Error: Cannot find module './zh-CN/common.json'
Error loading i18n file: ~/locales/zh-CN/auth.json: Error: Cannot find module './zh-CN/auth.json'
Error loading i18n file: ~/locales/zh-CN/profile.json: Error: Cannot find module './zh-CN/profile.json'
Error loading i18n file: ~/locales/zh-CN/subscription.json: Error: Cannot find module './zh-CN/subscription.json'
Error loading i18n file: ~/locales/zh-CN/organization.json: Error: Cannot find module './zh-CN/organization.json'
Error loading i18n file: ~/locales/zh-CN/onboarding.json: Error: Cannot find module './zh-CN/onboarding.json'
```

### 根本原因

1. **默认语言设置**: `configuration.ts` 中设置 `locale: 'zh-CN'`
2. **翻译文件缺失**: `public/locales/` 目录下只有 `en/` 目录，缺少 `zh-CN/` 目录
3. **i18n加载失败**: Next.js i18n系统无法加载zh-CN翻译，导致应用初始化失败
4. **请求超时**: 应用无法启动，所有HTTP请求超时

---

## 🎓 经验总结

### ✅ 成功经验

1. **完整翻译优于临时修复**:
   - 虽然改为英文能快速解决问题
   - 但创建完整翻译更符合长期目标

2. **保持结构一致性**:
   - 严格保持JSON结构与英文版一致
   - 避免翻译过程中引入结构错误

3. **使用自然的中文表达**:
   - 不是直译，而是符合中文习惯
   - 例如："Sign in" → "登录"，而非"签到"

4. **完整的文档记录**:
   - 详细记录翻译过程
   - 便于后续维护和扩展

### ⚠️ 注意事项

1. **占位符必须保留**:
   ```json
   // ❌ 错误
   "inviteUser": "邀请用户加入组织"

   // ✅ 正确
   "inviteUser": "邀请 {{email}} 加入 {{organization}}"
   ```

2. **HTML标签必须保留**:
   ```json
   // ❌ 错误
   "transferOwnership": "您正在将所有权转让给成员"

   // ✅ 正确
   "transferOwnership": "您正在将所有权转让给 <b>{{ member }}</b>"
   ```

3. **翻译键引用必须保留**:
   ```json
   // ❌ 错误
   "description": "您的新角色将是管理员"

   // ✅ 正确
   "description": "您的新角色将是 <b>$t(common:roles.admin.label)</b>"
   ```
