## 二、根本原因分析

### 2.1 Users表缺失

**问题**:
- Supabase只有 `auth.users` 表（存储认证信息）
- 缺少 `public.users` 表（存储业务数据）
- 触发器 `handle_new_user()` 尝试插入不存在的表

**证据**:
```sql
-- supabase/migrations/20250109_auto_create_user_simplified.sql (line 29-43)
INSERT INTO public.users (id, display_name, photo_url, onboarded, created_at)
VALUES (NEW.id, user_display_name, NEW.raw_user_meta_data->>'avatar_url', true, NOW())
ON CONFLICT (id) DO NOTHING;
```

**影响**:
- 新用户注册后无法创建业务记录
- 前端无法读取用户个人资料
- Navigation配置无法加载用户订阅信息

### 2.2 Navigation端点架构不匹配

**问题**:
- Console服务端点: `GET /api/v1/console/navigation/{organizationId}` (需要组织ID)
- 前端调用: `GET /console/navigation` (无参数，已移除组织概念)

**证据**:
```go
// services/console/internal/handlers/navigation.go (line 94-99)
orgUUID := strings.TrimSpace(r.PathValue("organizationId"))
if orgUUID == "" {
    statusLabel = "invalid_argument"
    pkgerrors.Write(w, r, http.StatusBadRequest, "INVALID_ARGUMENT",
                    "organizationId is required", nil)
    return
}
```

```typescript
// apps/frontend/src/lib/navigation/hooks.ts (line 13)
() => apiGet<NavigationResponse>(`/console/navigation`),
```

**影响**:
- 前端调用返回400 Bad Request
- 导航菜单无法渲染
- 用户被困在空白页面

### 2.3 API Gateway配置缺失

**问题1: 缺少 /console/navigation 路由**
```yaml
# deployments/gateway/gateway.v2.yaml
# ❌ 没有定义此路由
/api/v1/console/navigation:
```

**问题2: 无CORS配置**
```yaml
# gateway.v2.yaml 完全缺少 x-google-cors 配置
```

**影响**:
- 浏览器跨域请求被阻塞
- OPTIONS预检请求失败
- 所有API调用失败

## 三、修复方案

### 修复1: 创建public.users表 ✅ P0

**执行方式**: 在Supabase Dashboard SQL编辑器执行

**SQL脚本**:
```sql
-- Step 1: 创建users表
CREATE TABLE IF NOT EXISTS public.users (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    display_name TEXT,
    photo_url TEXT,
    onboarded BOOLEAN DEFAULT false,
    subscription_tier TEXT DEFAULT 'trial',
    monthly_token_allocation INTEGER DEFAULT 0,
    token_balance INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Step 2: 启用RLS
ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;

-- Step 3: 创建RLS策略 (用户只能读写自己的数据)
CREATE POLICY "Users can view own profile"
    ON public.users FOR SELECT
    USING (auth.uid() = id);

CREATE POLICY "Users can update own profile"
    ON public.users FOR UPDATE
    USING (auth.uid() = id);

-- Step 4: 为现有auth.users创建对应的public.users记录
INSERT INTO public.users (id, display_name, photo_url, onboarded)
SELECT
    id,
    COALESCE(raw_user_meta_data->>'full_name', raw_user_meta_data->>'name', email),
    raw_user_meta_data->>'avatar_url',
    true
FROM auth.users
ON CONFLICT (id) DO NOTHING;

-- Step 5: 验证
SELECT
    u.id,
    u.display_name,
    u.subscription_tier,
    u.token_balance,
    au.email
FROM public.users u
JOIN auth.users au ON au.id = u.id;
```

**预期结果**:
- 创建2条记录（对应现有2个auth.users）
- 触发器正常工作
- 前端可以读取用户信息

### 修复2: 重构Navigation端点为用户直连模式 ✅ P0

**方案A: 修改Console服务（推荐）**

移除 `organizationId` 参数依赖，改为直接基于 `user_id`：

```go
// services/console/internal/handlers/navigation.go

func (h *Handler) GetNavigationConfig(w http.ResponseWriter, r *http.Request) {
    // ... (认证逻辑不变)

    // ❌ 移除: 获取organizationId
    // orgUUID := strings.TrimSpace(r.PathValue("organizationId"))

    // ✅ 新增: 直接基于用户ID加载数据
    var (
        subscriptionTier  string
        monthlyAllocation int
        tokenBalance      int
    )

    if err := h.DB.QueryRow(ctx, `
        SELECT
            COALESCE(subscription_tier, 'trial')::text,
            COALESCE(monthly_token_allocation, 0),
            COALESCE(token_balance, 0)
        FROM users
        WHERE id = $1
    `, userID).Scan(&subscriptionTier, &monthlyAllocation, &tokenBalance); err != nil {
        if err != pgx.ErrNoRows {
            log.Printf("[navigation] failed to load user info: %v", err)
        }
        subscriptionTier = "trial"
        monthlyAllocation = 0
        tokenBalance = 0
    }

    // ✅ 构建简化导航（无组织概念）
    resp := NavigationResponse{
        SubscriptionTier:       subscriptionTier,
        TokenBalance:           tokenBalance,
        MonthlyTokenAllocation: monthlyAllocation,
        Items:                  buildSimpleNavigationItems(subscriptionTier),
        QuickActions:           buildSimpleQuickActions(subscriptionTier, tokenBalance),
        GeneratedAt:            time.Now(),
    }

    writeJSON(w, http.StatusOK, resp)
}

// 简化导航结构（无organizationId）
func buildSimpleNavigationItems(tier string) []NavigationItem {
    items := []NavigationItem{
        {Key: "dashboard", Type: "link", Label: "仪表盘", Path: "/dashboard", Icon: "dashboard"},
        {Key: "offers", Type: "link", Label: "Offers", Path: "/dashboard/offers", Icon: "offers"},
        {Key: "tasks", Type: "link", Label: "任务", Path: "/dashboard/tasks", Icon: "tasks"},
    }

    if allowAdsCenter(tier) {
        items = append(items, NavigationItem{
            Key: "ads", Type: "link", Label: "广告中心",
            Path: "/dashboard/ads-center", Icon: "ads",
        })
    }

    return items
}
```

**路由注册调整**:
