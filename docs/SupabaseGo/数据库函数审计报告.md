## 🔍 审计方法

### SQL 审计查询

```sql
-- 1. 查询所有包含 "organization" 的函数
SELECT
  routine_name,
  routine_type,
  routine_definition
FROM information_schema.routines
WHERE routine_schema = 'public'
  AND (
    routine_name LIKE '%organization%'
    OR routine_name LIKE '%org%'
    OR routine_name LIKE '%member%'
    OR routine_name LIKE '%invitation%'
  );

-- 2. 查询函数参数
SELECT
  routine_name,
  parameter_name,
  data_type
FROM information_schema.parameters
WHERE specific_schema = 'public'
  AND (
    routine_name LIKE '%organization%'
    OR parameter_name LIKE '%organization%'
  )
ORDER BY routine_name, ordinal_position;
```

## 🗑️ 删除建议

### 高优先级删除（P0 - 必须删除）

以下函数直接与组织功能相关，必须删除：

```sql
-- 组织 CRUD
DROP FUNCTION IF EXISTS get_user_organizations(uuid);
DROP FUNCTION IF EXISTS create_organization(text, uuid);
DROP FUNCTION IF EXISTS update_organization(uuid, text);
DROP FUNCTION IF EXISTS delete_organization(uuid);
DROP FUNCTION IF EXISTS get_organization_by_uid(text);

-- 成员管理
DROP FUNCTION IF EXISTS get_organization_members(uuid);
DROP FUNCTION IF EXISTS invite_organization_member(uuid, text, text);
DROP FUNCTION IF EXISTS remove_organization_member(uuid, uuid);
DROP FUNCTION IF EXISTS update_member_role(uuid, uuid, text);

-- 邀请管理
DROP FUNCTION IF EXISTS accept_organization_invitation(uuid);
DROP FUNCTION IF EXISTS reject_organization_invitation(uuid);
DROP FUNCTION IF EXISTS get_pending_invitations(uuid);

-- 权限检查
DROP FUNCTION IF EXISTS check_organization_permission(uuid, uuid);
DROP FUNCTION IF EXISTS get_user_role_in_organization(uuid, uuid);
```

### 中优先级审查（P1 - 需要确认）

以下函数可能间接使用组织概念，需要逐个审查：

- `get_*_by_organization_id()` - 所有通过 `organization_id` 查询的函数
- `organization_*` - 所有以 organization 开头的函数

### 低优先级保留（P2 - 暂不删除）

以下类型的函数暂不删除：

- 用户相关函数（已改为 `user_id` 参数）
- 订阅相关函数（已改为用户级订阅）
- 通用工具函数
