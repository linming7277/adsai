### 4. 查询超时控制工具

**文件**: `pkg/database/context.go`

**提供的超时级别**:

```go
// 默认超时（10秒）
ctx, cancel := database.WithQueryTimeout(ctx, 0)
defer cancel()

// 快速查询（5秒）- 简单的SELECT
ctx, cancel := database.WithShortQueryTimeout(ctx)

// 中等查询（30秒）- 复杂查询或批量操作
ctx, cancel := database.WithMediumQueryTimeout(ctx)

// 长查询（60秒）- 重型分析或大批量操作
ctx, cancel := database.WithLongQueryTimeout(ctx)
```

**使用示例**:
```go
func (r *OfferRepository) ListByUser(ctx context.Context, userID string) ([]*Offer, error) {
    // 添加5秒超时，防止单个用户的慢查询影响其他用户
    ctx, cancel := database.WithShortQueryTimeout(ctx)
    defer cancel()

    rows, err := r.db.QueryContext(ctx, `
        SELECT id, name, status, "createdAt"
        FROM "Offer"
        WHERE "userId" = $1
        ORDER BY "createdAt" DESC
        LIMIT 100
    `, userID)

    // ... 处理结果
}
```

## ✅ 索引迁移执行总结

### 执行过程

由于本地无法直接访问VPC内网数据库，采用了以下解决方案：

1. **创建Go迁移程序**: 使用Go的`lib/pq`驱动，能正确处理URL编码的密码
2. **Docker化**: 构建轻量级Docker镜像（基于distroless）
3. **Cloud Run Job**: 通过VPC Connector访问数据库
4. **自动化部署**: Cloud Build自动构建+部署+执行

### 技术挑战与解决

| 挑战 | 原因 | 解决方案 |
|------|------|----------|
| DATABASE_URL解析失败 | 密码包含特殊字符`@` | 使用Go的pq驱动正确解析 |
| 表名不匹配 | 文档使用大写，实际是小写 | 检查实际schema并修正 |
| VPC访问 | 本地无法连接内网IP | Cloud Run Job + VPC Connector |
