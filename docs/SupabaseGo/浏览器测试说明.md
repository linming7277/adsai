### 方法2: Playwright自动化测试

**如果使用Playwright MCP**，需要测试：

#### 1. 公开页面测试
```javascript
// 访问首页
await page.goto('https://preview.example.com');

// 验证页面加载
await expect(page).toHaveTitle(/AdsAI|URL Checker/);

// 检查登录按钮
await expect(page.locator('text=Sign in')).toBeVisible();
```

#### 2. 登录流程测试
```javascript
// 点击登录按钮
await page.click('text=Sign in with Google');

// 等待Google OAuth页面
await page.waitForURL('**/accounts.google.com/**');

// 注意：Google OAuth需要真实账号，难以自动化

```

#### 3. 认证后路由测试
```javascript
// 假设已登录，访问Dashboard
await page.goto('https://preview.example.com/dashboard/offers');

// 验证页面加载
await expect(page.locator('h1')).toContainText('Offers');

// 验证数据显示
await expect(page.locator('table')).toBeVisible();
```

#### 4. RBAC测试
```javascript
// 检查管理员专属菜单
await expect(page.locator('text=后台管理')).toBeVisible();

// 点击Admin链接
await page.click('text=Admin');

// 验证跳转到/manage
await expect(page).toHaveURL(/.*\/manage/);
```

#### 5. 导航测试
```javascript
// 点击侧边栏链接
const navLinks = [
  'Offer 管理',
  '任务中心',
  '广告中心',
  '系统设置'
];

for (const link of navLinks) {
  await page.click(`text=${link}`);
  await page.waitForLoadState('networkidle');
  // 验证无404错误
  const content = await page.content();
  expect(content).not.toContain('404');
}
```

## 🔧 Playwright MCP局限性

### 已知问题
1. **Google OAuth难以自动化**: 需要真实Google账号授权
2. **Session管理复杂**: 需要手动提取和注入Session Token
3. **Cloudflare Challenge**: 可能触发人机验证

### 解决方案

#### A. Session Token注入（推荐）
```javascript
// 1. 手动登录一次，从浏览器获取Token
// 2. 注入到Playwright测试中
await page.context().addCookies([{
  name: 'sb-access-token',
  value: 'YOUR_SUPABASE_TOKEN',
  domain: '.preview.example.com',
  path: '/',
  httpOnly: true,
  secure: true
}]);

// 3. 刷新页面，应该已登录
await page.reload();
```

#### B. 使用测试专用账号
- 创建专门的测试账号
- 设置简化的登录流程（仅用于测试环境）
- 存储Session Token在环境变量

### 如果你有Playwright MCP工具

1. **安装依赖**
   ```bash
   npm install -g @playwright/test
   playwright install chromium
   ```

2. **创建测试脚本**
   ```bash
   mkdir -p tests/e2e
   touch tests/e2e/preview-env.spec.js
   ```

3. **编写测试**（参考上面的代码示例）

4. **运行测试**
