## 🔧 修复策略

### 策略1: 参考Dashboard概览页面（已通过测试）

Dashboard概览页面测试通过，说明其实现是正确的。其他页面应参考该页面的实现方式。

**Dashboard概览页面的成功要素**:
1. ✅ 正确添加了`data-testid`属性
   ```typescript
   <div data-testid="stat-card-total-offers">
   ```
2. ✅ 统计卡片正常渲染（4/4）
3. ✅ 快速操作按钮正常显示（5/5）
4. ✅ 条件渲染逻辑合理

**参考文件**:
- `apps/frontend/app/dashboard/page.tsx`
- 相关组件: `StatCard`、`QuickActions`等

---

### 策略2: 统一添加data-testid属性

为所有测试相关的UI组件添加`data-testid`属性，遵循命名规范：

**命名规范**:
```
{page}-{component-type}-{identifier}

示例:
- token-tile-balance (Token页面 + 卡片 + 余额)
- task-tab-running (任务页面 + Tab + 进行中)
- ads-center-stat-card-spend (广告中心 + 统计卡片 + 支出)
```

---

### 策略3: 简化条件渲染逻辑

**错误示例**（过于严格）:
```typescript
{userData && userData.subscription && userData.tokens > 0 && (
  <StatsCard balance={userData.tokens} />
)}
```

**正确示例**（宽松且有fallback）:
```typescript
{userData ? (
  <StatsCard balance={userData.tokens || 0} />
) : (
  <Skeleton /> // Loading状态
)}
```

---

### 策略4: 确保API数据正常加载

检查Console API端点是否返回正确数据：

**需要检查的API**:
- `GET /api/console/users/me`
- `GET /api/console/tokens`
- `GET /api/console/tasks`
- `GET /api/console/ads-accounts`
- `GET /api/console/subscriptions/current`

**验证方法**:
1. 打开浏览器开发者工具Network标签
2. 访问失败的页面
3. 检查API请求状态码（应该是200 OK）
4. 查看响应内容（应该有完整的数据，不是空对象）
