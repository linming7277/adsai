## 🔍 根本原因分析

### 可能原因1: 缺少data-testid属性

**证据**:
- 测试脚本通过`data-testid`定位元素
- 所有失败的测试都无法找到带有特定`data-testid`的元素

**验证方法**:
1. 手动访问 https://www.urlchecker.dev/settings/tokens
2. 打开浏览器开发者工具
3. 检查统计卡片是否有`data-testid="token-tile-balance"`等属性
4. 如果属性缺失，需要在组件中添加

**修复方案**:
- 在所有测试相关的UI组件上添加`data-testid`属性
- 参考已通过的Dashboard概览测试（已正确添加data-testid）

---

### 可能原因2: 条件渲染逻辑过于严格

**证据**:
- Dashboard概览测试通过（说明基础认证正常）
- 但Dashboard的子页面大量组件不渲染

**可能的条件检查**:
```typescript
// 示例：过于严格的条件渲染
{userData && userData.subscription && userData.tokens > 0 && (
  <StatsCard />
)}

// 应该改为更宽松的条件
{userData && (
  <StatsCard />
)}
```

**验证方法**:
1. 检查组件的条件渲染逻辑（`&&`、三元运算符、`if`语句）
2. 查看是否依赖多个条件同时满足
3. 添加日志输出条件检查结果

**修复方案**:
- 简化条件渲染逻辑
- 添加loading和error状态的fallback UI
- 确保在数据加载失败时也显示骨架屏或空状态

---

### 可能原因3: Console API数据加载失败

**证据**:
- 统计卡片依赖后端API数据
- 如果API返回空或错误，组件可能不渲染

**需要检查的API**:
- `GET /api/console/users/me` - 用户信息
- `GET /api/console/tokens` - Token余额
- `GET /api/console/tasks` - 任务列表
- `GET /api/console/ads-accounts` - 广告账户
- `GET /api/console/subscriptions/current` - 当前订阅

**验证方法**:
1. 打开浏览器开发者工具Network标签
2. 访问失败的页面
3. 检查API请求状态（200 OK？404？500？）
4. 查看API响应内容（是否为空？格式是否正确？）

**修复方案**:
- 修复返回错误的API端点
- 确保测试用户有完整的种子数据
- 添加API错误处理和重试逻辑
- 显示友好的错误提示而非空白页面

---

### 可能原因4: CSS/布局问题

**证据**:
- 组件可能渲染了，但由于CSS问题不可见

**可能的CSS问题**:
```css
/* 错误的z-index */
.stats-card {
  z-index: -1; /* 被其他元素遮挡 */
}

/* 错误的display */
.stats-card {
  display: none; /* 或 visibility: hidden */
}

/* 错误的尺寸 */
.stats-card {
  width: 0;
  height: 0;
  overflow: hidden;
}
```

**验证方法**:
1. 在浏览器开发者工具Elements标签中搜索组件
2. 如果找到DOM节点，检查computed styles
3. 查看是否有`display: none`、`opacity: 0`、`visibility: hidden`等

**修复方案**:
- 检查并修复CSS样式
- 使用浏览器开发者工具实时调试样式
- 确保响应式布局在所有屏幕尺寸下正常

---

### 可能原因5: 国际化(i18n)问题

**证据**:
- 测试脚本查找按钮时使用文本匹配（如"充值"、"购买 Token"）
- 如果i18n配置错误，文本可能不匹配

**可能的i18n问题**:
```typescript
// 错误：硬编码中文
<button>充值</button>

// 正确：使用t()函数
<button>{t('tokens.recharge')}</button>
```

**验证方法**:
1. 检查按钮和标签是否使用`t()`函数
2. 查看i18n配置文件是否有对应的key
3. 测试中英文切换是否正常

**修复方案**:
- 确保所有用户可见文本使用`t()`函数
- 补全缺失的翻译key
- 在测试脚本中使用更灵活的选择器（如data-testid而非文本）
