## 🎯 核心概念澄清

### Offer 生命周期（更新后）

```
1. 添加 Offer → 2. 评估 → 3. 补点击（可选）→ 4. 投放
   粘贴 URL      10秒评分    配置任务参数      关联 Ads 账号

关键理解：
✅ 评估: 10秒智能评估，得到 0-100 分
✅ 补点击: 独立的流量获取操作，与 Ads 账号无关
   - 配置参数: 仅需目标点击数 + 投放时长
   - 显示数据: 仅显示点击数 / 花费
   - 不产生: ROAS、收入、转化等业务数据
✅ 投放: 建立 Offer ←→ Ads Account 关联关系
   - 选择 Ads 账号（可多选）
   - 开始产生 ROAS 等业务数据
```

### 3. 数据库表结构（03-数据库表结构.md）

**主要调整**:

1. **click_tasks 表（补点击任务）**
   - ❌ 删除字段:
     - `ads_account_ids` JSONB（补点击与 Ads 账号无关）
     - `revenue` DECIMAL
     - `conversions` INTEGER
     - `roas` DECIMAL

   - ✅ 保留字段:
     - `target_clicks` INTEGER（目标点击数）
     - `completed_clicks` INTEGER（已完成点击数）
     - `spend` DECIMAL（花费金额）
     - `duration_hours` INTEGER（投放时长）

2. **offers 表**
   - ✅ 新增字段:
     - `deployment_status` VARCHAR(20)（投放状态: PENDING / DEPLOYED）

3. **offer_ads_links 表（新增）**
   - ✅ 用于存储 Offer ←→ Ads Account 关联关系
   ```sql
   CREATE TABLE offer_ads_links (
     id VARCHAR(36) PRIMARY KEY,
     offer_id VARCHAR(36) REFERENCES offers(id),
     ads_account_id VARCHAR(36) REFERENCES ads_accounts(id),
     created_at TIMESTAMP WITH TIME ZONE,
     UNIQUE(offer_id, ads_account_id)
   );
   ```

### 5. API 接口设计（05-API接口设计.md）

**主要调整**:

1. **补点击接口修正**

   **原接口（错误）:**
   ```http
   POST /api/v1/batchopen/simulate
   {
     "offer_id": "offer_abc123",
     "ads_account_ids": ["ads_123", "ads_456"],  // ❌ 删除
     "simulation_config": { ... }
   }
   ```

   **新接口（正确）:**
   ```http
   POST /api/v1/batchopen/click-tasks
   {
     "offer_id": "offer_abc123",
     "target_clicks": 100,
     "duration_hours": 24
   }
   ```

2. **补点击进度查询**

   **Response（更新后）:**
   ```json
   {
     "task_id": "click_task_xyz",
     "status": "IN_PROGRESS",
     "target_clicks": 100,
     "completed_clicks": 45,
     "spend": 23.50,
     // ❌ 删除: revenue, conversions, roas
   }
   ```

3. **新增投放接口**

   ```http
   POST /api/v1/offers/{offer_id}/deploy
   {
     "ads_account_ids": ["ads_123", "ads_456"]
   }
   ```

   **Response:**
   ```json
   {
     "offer_id": "offer_abc123",
     "deployment_status": "DEPLOYED",
     "linked_accounts": [
       {
         "id": "ads_123",
         "account_name": "Nike US 1",
         "status": "ACTIVE"
       }
     ]
   }
   ```

## 🔄 数据流转澄清

### 补点击阶段 vs 投放阶段

```
补点击阶段:
- 触发: 用户手动点击"启动补点击"
- 配置: 目标点击数 + 投放时长
- 执行: browser-exec 服务自动执行
- 数据: 点击数、花费（无 ROAS）
- 状态: click_task_status

投放阶段:
- 触发: 用户手动点击"投放到账号"
- 配置: 选择 Ads 账号（可多选）
- 执行: 建立 Offer ←→ Ads Account 关联关系
- 数据: ROAS、收入、转化（真实业务数据）
- 状态: deployment_status
```
