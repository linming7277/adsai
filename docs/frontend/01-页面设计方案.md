# AdsAI 前端页面设计方案

> 基于"上瘾式广告管理系统"需求，设计精简、易用的前端页面
>
> 版本: v2.0
> 更新时间: 2024-10-03
> 设计原则: 精简操作、即时反馈、智能化、可视化
>
> **v2.0 更新内容:**
> - 术语修正: "仿真" → "补点击"
> - 流程简化: 删除自动化规则，改为手动触发
> - 组件复用: 基于 Makerkit + Radix UI 组件库
> - 数据精简: 补点击阶段不显示 ROAS 等业务数据

---

## 📐 整体架构

### 页面结构总览

```
顶部导航栏（固定）
┌────────────────────────────────────────────────────────────┐
│ [Logo] AdsAI  [大盘] [Offer库] [Ads中心] [任务中心]     │
│                          [🔔通知3] [🪙Token:1250] [👤用户]  │
└────────────────────────────────────────────────────────────┘

核心页面路由：
/dashboard   - 大盘（首页，日常运营概览）
/offers      - Offer库（核心业务页面）
/adscenter   - Ads中心（账号管理）
/tasks       - 任务中心（Token消耗透明化）

辅助功能：
侧边栏/弹窗 - 个人中心（套餐、Token、签到、邀请）
```

### 设计理念

1. **精简至上** - 每个页面聚焦1-2个核心任务
2. **即时反馈** - 操作后<0.5秒显示反馈
3. **用户掌控** - 关键操作由用户主动触发，避免自动化
4. **可视化** - 复杂数据用图表和健康度简化

### 核心流程（Offer 生命周期）

```
1. 添加 Offer → 2. 评估 → 3. 补点击（可选）→ 4. 投放
   粘贴 URL      10秒评分    配置任务参数      关联 Ads 账号

说明：
- 评估: 10秒智能评估，得到 0-100 分
- 补点击: 独立的流量获取操作，与 Ads 账号无关，仅显示点击数/花费
- 投放: 建立 Offer ←→ Ads Account 关联关系，开始产生 ROAS 等业务数据
```

### 核心功能模块

#### 1. 风险提醒区
**功能**: 自动检测并展示需要关注的问题

**风险等级定义**:
| 等级 | 图标 | 触发条件 | 自动建议 |
|-----|------|---------|---------|
| 🔴 紧急 | 红色 | • 账号余额<$20<br>• ROAS<0.5且花费>$100 | 提供一键修复 |
| 🟡 警告 | 黄色 | • ROAS连续3天下滑>20%<br>• CTR<1%连续2天 | 批量诊断 |
| 🔵 提醒 | 蓝色 | • 新Offer待评估<br>• 任务完成通知 | 快捷跳转 |

**一键修复示例**:
```
🔴 Ads账号预算不足
影响: Nike US 2、Survey US 3
详情: 2个账号余额低于$20，预计6小时后暂停广告

💡 建议方案:
• 从 Nike US 1 ($250) 转移 $50 到 Nike US 2
• 从 Adidas UK 1 ($180) 转移 $50 到 Survey US 3

[一键执行转移] [手动处理] [稍后提醒]
```

#### 2. 核心指标概览
**功能**: 显示最重要的4个指标

**指标说明**:
- **Offer总数**: 显示总数和本周新增
- **Ads账号数**: 显示总数和状态分布
- **今日花费**: 显示金额和对比昨日增长率
- **平均ROAS**: 显示数值和趋势变化

**可自定义指标**:
- Token余额与消耗
- 总曝光量/总点击量
- 平均CPC/平均CTR
- 本月总收入/总花费

#### 3. Top Offer 排行
**智能排序逻辑**:
1. 紧急问题 Offer 优先（ROAS<1且花费>$50）
2. 高价值 Offer（ROAS>2且花费>$100）
3. 新评估完成的 Offer
4. 最近有状态变化的 Offer

**健康度评分规则（简化为三色状态灯）**:
- 🟢 健康: ROAS ≥ 2.0
- 🟡 警告: 1.5 ≤ ROAS < 2.0
- 🔴 异常: ROAS < 1.5（建议暂停）

**注意**: 仅投放阶段的 Offer 显示健康度，补点击阶段显示"补点击中"

#### 4. 趋势图表
**显示内容**:
- 折线图: 花费、收入、ROAS
- 时间范围: 7天/30天/90天
- 关键洞察: 自动分析趋势变化

**交互功能**:
- 鼠标悬停显示详细数据
- 点击图例切换显示/隐藏
- 下载报表、分享功能

### 页面间联动

**大盘 → Offer库**:
```
点击"暂停 Casino Offer"

→ 执行暂停

```

#### 6. 批量操作

**选中多个 Offer 后显示操作栏**:
```
┌───────────────────────────────────────────────────────┐
│  ✅ 已选择 3 个 Offer                     [取消选择]  │

│  [批量评估] [批量补点击] [批量投放] [批量修改国家]   │

**批量操作说明**:
- **批量评估**: 对所有选中的 Offer 执行评估
- **批量补点击**: 对已评估的 Offer 统一配置补点击任务

// 从 0 滚动到目标值
function animateNumber(element, target, duration) {
  const start = 0;
  const increment = (target - start) / (duration / 16);
  let current = start;

  const timer = setInterval(() => {
    current += increment;
    if (current >= target) {
      current = target;
      clearInterval(timer);
    }
    element.textContent = Math.round(current);
  }, 16);
}
```
