# AdsAI 数据库表结构设计

> 基于 PostgreSQL + Firestore 的混合数据存储方案
>
> 版本: v2.0
> 更新时间: 2024-10-03
>
> **v2.0 更新内容:**
> - 修正 click_tasks 表（删除 ads_account_ids、revenue、conversions、roas）
> - 新增 offer_ads_links 表（存储 Offer ←→ Ads Account 关联关系）
> - 新增 offers 表的 deployment_status 字段

---

## 📊 数据存储策略

### 存储分层

| 存储类型 | 技术选型 | 用途 | 特点 |
|---------|---------|------|------|
| **关系型数据库** | PostgreSQL | 核心业务数据、历史数据、复杂查询 | ACID事务、关联查询、数据一致性 |
| **文档数据库** | Firestore | 实时缓存、UI状态、配置数据 | 实时同步、无模式、快速读写 |
| **缓存层** | Redis | 热点数据、会话管理 | 高性能、过期策略 |

### 数据分配原则

**PostgreSQL 存储**:
- ✅ Offer 核心数据
- ✅ Ads 账号信息
- ✅ 任务执行记录
- ✅ 用户订阅、Token 交易
- ✅ 审计日志

**Firestore 存储**:
- ✅ 用户 UI 偏好设置
- ✅ 实时通知消息
- ✅ 系统配置（评估规则、预警阈值）
- ✅ 临时数据缓存

**Redis 存储**:
- ✅ Session 数据
- ✅ 热点 Offer 数据
- ✅ API 限流计数

### 1. 用户相关表

#### 1.1 users（用户表）

```sql
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  firebase_uid VARCHAR(128) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  display_name VARCHAR(100),
  avatar_url VARCHAR(500),
  role VARCHAR(20) DEFAULT 'USER' CHECK (role IN ('USER', 'ADMIN')),
  status VARCHAR(20) DEFAULT 'ACTIVE' CHECK (status IN ('ACTIVE', 'SUSPENDED', 'DELETED')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  last_login_at TIMESTAMP WITH TIME ZONE
);

-- 索引
CREATE INDEX idx_users_firebase_uid ON users(firebase_uid);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);

-- 注释
COMMENT ON TABLE users IS '用户基础信息表';
COMMENT ON COLUMN users.firebase_uid IS 'Firebase Authentication UID';
COMMENT ON COLUMN users.role IS '用户角色：USER-普通用户, ADMIN-管理员';
COMMENT ON COLUMN users.status IS '用户状态：ACTIVE-活跃, SUSPENDED-暂停, DELETED-已删除';
```

### 2. Offer相关表

#### 2.1 offers（Offer表）

```sql
CREATE TABLE offers (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36) NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  offer_url TEXT NOT NULL,
  final_url TEXT,
  final_url_suffix TEXT,
  brand_name VARCHAR(100),
  country_code VARCHAR(2) NOT NULL DEFAULT 'US',

  -- 评估数据
  evaluation_status VARCHAR(20) DEFAULT 'PENDING' CHECK (
    evaluation_status IN ('PENDING', 'EVALUATING', 'COMPLETED', 'FAILED')
  ),
  evaluation_score INTEGER CHECK (evaluation_score >= 0 AND evaluation_score <= 100),
  evaluation_result JSONB,
  evaluated_at TIMESTAMP WITH TIME ZONE,

  -- 补点击数据（可选阶段）
  click_task_status VARCHAR(20) DEFAULT 'PENDING' CHECK (
    click_task_status IN ('PENDING', 'RUNNING', 'COMPLETED', 'FAILED')
  ),
  click_task_id VARCHAR(36),
  click_task_completed_at TIMESTAMP WITH TIME ZONE,

  -- 投放数据
  deployment_status VARCHAR(20) DEFAULT 'PENDING' CHECK (
    deployment_status IN ('PENDING', 'DEPLOYED', 'PAUSED', 'ARCHIVED')
  ),
  deployed_at TIMESTAMP WITH TIME ZONE,

  -- 性能指标
  impressions BIGINT DEFAULT 0,
  clicks BIGINT DEFAULT 0,
  ctr DECIMAL(5, 2) DEFAULT 0,
  avg_cpc DECIMAL(10, 2) DEFAULT 0,
  total_spend DECIMAL(10, 2) DEFAULT 0,
  total_revenue DECIMAL(10, 2) DEFAULT 0,
  roas DECIMAL(10, 2) DEFAULT 0,

  -- 健康度
  health_score INTEGER CHECK (health_score >= 0 AND health_score <= 100),
  last_health_check_at TIMESTAMP WITH TIME ZONE,

  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  archived_at TIMESTAMP WITH TIME ZONE
);

-- 索引
CREATE INDEX idx_offers_user_id ON offers(user_id);
CREATE INDEX idx_offers_evaluation_status ON offers(evaluation_status);
CREATE INDEX idx_offers_click_task_status ON offers(click_task_status);
CREATE INDEX idx_offers_deployment_status ON offers(deployment_status);
CREATE INDEX idx_offers_roas ON offers(roas);
CREATE INDEX idx_offers_created_at ON offers(created_at DESC);

-- 全文搜索索引
CREATE INDEX idx_offers_brand_name ON offers USING gin(to_tsvector('simple', brand_name));

-- 注释
COMMENT ON TABLE offers IS 'Offer核心数据表';
COMMENT ON COLUMN offers.offer_url IS '广告联盟提供的Offer URL';
COMMENT ON COLUMN offers.final_url IS '最终落地页URL';
COMMENT ON COLUMN offers.evaluation_result IS '评估结果JSON（流量潜力、关键词等）';
COMMENT ON COLUMN offers.click_task_status IS '补点击任务状态';
COMMENT ON COLUMN offers.click_task_id IS '关联的补点击任务ID';
COMMENT ON COLUMN offers.deployment_status IS '投放状态：PENDING-未投放, DEPLOYED-已投放, PAUSED-暂停, ARCHIVED-归档';
```

CREATE TABLE tasks (

#### 4.3 click_tasks（补点击任务表）

```sql
CREATE TABLE click_tasks (
  id VARCHAR(36) PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id VARCHAR(36) NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  offer_id VARCHAR(36) NOT NULL REFERENCES offers(id) ON DELETE CASCADE,

  -- 配置参数（仅2个）
  target_clicks INTEGER NOT NULL CHECK (target_clicks > 0),
  duration_hours INTEGER NOT NULL CHECK (duration_hours > 0),

  -- 执行状态
  status VARCHAR(20) DEFAULT 'QUEUED' CHECK (
    status IN ('QUEUED', 'RUNNING', 'COMPLETED', 'FAILED', 'CANCELLED')
  ),
  completed_clicks INTEGER DEFAULT 0 CHECK (completed_clicks >= 0),
  spend DECIMAL(10, 2) DEFAULT 0 CHECK (spend >= 0),

  -- Token消耗
  token_cost INTEGER NOT NULL DEFAULT 0,
  token_consumed INTEGER NOT NULL DEFAULT 0,

  -- 时间记录
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  started_at TIMESTAMP WITH TIME ZONE,
  completed_at TIMESTAMP WITH TIME ZONE,

  -- 执行环境
  worker_id VARCHAR(50),
  error_message TEXT
);

-- 索引
CREATE INDEX idx_click_tasks_user_id ON click_tasks(user_id);
CREATE INDEX idx_click_tasks_offer_id ON click_tasks(offer_id);
CREATE INDEX idx_click_tasks_status ON click_tasks(status);
CREATE INDEX idx_click_tasks_created_at ON click_tasks(created_at DESC);

-- 注释
COMMENT ON TABLE click_tasks IS '补点击任务表';
COMMENT ON COLUMN click_tasks.target_clicks IS '目标点击数（用户配置）';
COMMENT ON COLUMN click_tasks.duration_hours IS '投放时长（小时）';
COMMENT ON COLUMN click_tasks.completed_clicks IS '已完成点击数';
COMMENT ON COLUMN click_tasks.spend IS '花费金额（仅显示点击成本，无ROAS数据）';
COMMENT ON COLUMN click_tasks.status IS '任务状态：QUEUED-排队, RUNNING-执行中, COMPLETED-已完成, FAILED-失败, CANCELLED-已取消';
```

**关键说明**:
- ❌ **不包含** `ads_account_ids` - 补点击与 Ads 账号无关
- ❌ **不包含** `revenue`、`conversions`、`roas` - 补点击不产生业务数据
- ✅ **仅包含** 点击数、花费等基础执行数据
- ✅ 配置参数仅需 2 个: `target_clicks` + `duration_hours`

-- Offer更新时同步到Firestore
CREATE OR REPLACE FUNCTION sync_offer_to_firestore()
RETURNS TRIGGER AS $$
BEGIN
  -- 发布到 Pub/Sub Topic
  PERFORM pg_notify(
    'firestore_sync',
    json_build_object(
      'collection', 'offers',
      'docId', NEW.id,
      'operation', TG_OP,
      'data', row_to_json(NEW)
    )::text
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER offer_sync_trigger
AFTER INSERT OR UPDATE ON offers
FOR EACH ROW
EXECUTE FUNCTION sync_offer_to_firestore();

```typescript
import crypto from 'crypto';

// 加密
function encryptRefreshToken(token: string): string {
  const key = process.env.ENCRYPTION_KEY!;
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv('aes-256-gcm', Buffer.from(key, 'hex'), iv);

  let encrypted = cipher.update(token, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  const authTag = cipher.getAuthTag();

  return `${iv.toString('hex')}:${authTag.toString('hex')}:${encrypted}`;
}

// 解密
function decryptRefreshToken(encrypted: string): string {
  const key = process.env.ENCRYPTION_KEY!;
  const [ivHex, authTagHex, encryptedText] = encrypted.split(':');

  const decipher = crypto.createDecipheriv(
    'aes-256-gcm',
    Buffer.from(key, 'hex'),
    Buffer.from(ivHex, 'hex')
  );

  decipher.setAuthTag(Buffer.from(authTagHex, 'hex'));

  let decrypted = decipher.update(encryptedText, 'hex', 'utf8');
  decrypted += decipher.final('utf8');

  return decrypted;
}
```
