## 🗂️ 路由架构说明

### 1. 后端API路由（Go微服务）

这些是**后端微服务的API端点**，通过API Gateway暴露：

```
API Gateway Base URL: https://autoads-gw-preview-885pd7lz.an.gateway.dev

后端API路由（微服务）:
├── /api/v1/offers          - Offer服务
├── /api/v1/siterank        - SiteRank评估服务
├── /api/v1/adscenter       - Google Ads管理服务
├── /api/v1/batchopen       - 批量访问服务
├── /api/v1/billing         - 计费服务
├── /api/v1/tasks           - 任务服务
└── /api/v1/notifications   - 通知服务
```

**说明**:
- ✅ 这些是**后端Go微服务的API路径**
- ✅ 前端通过API调用这些服务
- ❌ **不一定对应前端页面路由**

---

### 2. 前端页面路由（Next.js Pages）

根据《01-页面设计方案.md》，实际的前端页面只有**4个核心页面** + **1个辅助功能**：

```
前端页面路由（Next.js）:
├── /dashboard         - 页面一：大盘（运营概览）
├── /offers            - 页面二：Offer库（核心业务）
├── /adscenter         - 页面三：Ads中心（账号管理）
├── /tasks             - 页面四：任务中心（Token透明化）
└── /settings/*        - 个人中心（侧边栏/弹窗，或独立页面）
```

**说明**:
- ✅ 这些是**真实的前端页面**，用户可访问
- ✅ 每个页面对应《01-页面设计方案.md》中的设计
- ✅ 页面内部会调用多个后端API服务

## 📋 实际前端页面清单

根据《01-页面设计方案.md》，需要实现的前端页面：

### 核心页面（4个）

#### 1. 大盘页面 `/dashboard`

**文件**: `apps/frontend/src/pages/dashboard/index.tsx`

**功能**:
- 运营概览（核心指标、风险提醒）
- Top Offer数据展示
- 快速操作入口

**调用的后端API**:
- `GET /api/v1/dashboard/overview`
- `GET /api/v1/dashboard/trends`
- `GET /api/v1/notifications/unread-count`

---

#### 2. Offer库页面 `/offers`

**文件**: `apps/frontend/src/pages/offers/index.tsx`

**功能**:
- Offer列表（TanStack Table）
- 添加Offer弹窗
- 评估功能
- 补点击配置
- 投放配置

**调用的后端API**:
- `GET /api/v1/offers` - 列表
- `POST /api/v1/offers` - 添加
- `POST /api/v1/siterank/evaluate` - 评估（SiteRank服务）
- `POST /api/v1/batchopen/click-tasks` - 补点击（BatchOpen服务）
- `POST /api/v1/offers/{id}/deploy` - 投放

**关键理解**:
- ❌ **没有** `/siterank` 和 `/batchopen` 前端页面
- ✅ 这些功能都**集成在 `/offers` 页面中**
- ✅ 通过弹窗、按钮触发调用后端API

---

#### 3. Ads中心页面 `/adscenter`

**文件**: `apps/frontend/src/pages/adscenter/index.tsx`

**功能**:
- Google Ads账号列表
- OAuth授权连接
- 账号同步
- 预算转移

**调用的后端API**:
- `GET /api/v1/adscenter/oauth/url` - 获取OAuth URL
- `POST /api/v1/adscenter/oauth/callback` - 处理回调
- `GET /api/v1/adscenter/accounts` - 账号列表
- `POST /api/v1/adscenter/accounts/{id}/sync` - 同步

**OAuth回调页面**:
- **文件**: `apps/frontend/src/pages/adscenter/callback.tsx`
- **路由**: `/adscenter/callback`
- **用途**: 处理Google OAuth回调

---

#### 4. 任务中心页面 `/tasks`

**文件**: `apps/frontend/src/pages/tasks/index.tsx`

**功能**:
- 任务列表（评估、补点击、部署等）
- 任务进度查询
- 任务取消

**调用的后端API**:
- `GET /api/v1/tasks` - 列表
- `GET /api/v1/tasks/{id}` - 详情
- `POST /api/v1/tasks/{id}/cancel` - 取消

## 🔄 前端页面与后端API的映射关系

### Offer库页面 `/offers` 的API调用

| 功能 | 用户操作 | 调用的后端API | 后端服务 |
|------|---------|--------------|----------|
| **添加Offer** | 粘贴URL，点击"添加" | `POST /api/v1/offers` | Offer服务 |
| **评估Offer** | 点击"评估"按钮 | `POST /api/v1/siterank/evaluate` | SiteRank服务 |
| **补点击** | 点击"启动补点击" | `POST /api/v1/batchopen/click-tasks` | BatchOpen服务 |
| **投放** | 点击"投放到账号" | `POST /api/v1/offers/{id}/deploy` | Offer服务 |
| **查看列表** | 页面加载 | `GET /api/v1/offers` | Offer服务 |

**关键点**:
- ✅ `/offers` 页面是前端页面路由
- ✅ `/api/v1/siterank` 和 `/api/v1/batchopen` 是后端API路由
- ✅ **用户不会访问 `/siterank` 或 `/batchopen` 前端页面**
- ✅ 所有操作都在 `/offers` 页面通过弹窗/按钮完成

### 正确的OAuth回调配置

**前端页面路由**: `/adscenter`

**Google Cloud Console配置**:
```
Authorized redirect URIs:
https://www.urlchecker.dev/adscenter/callback
https://www.autoads.dev/adscenter/callback
http://localhost:3000/adscenter/callback
```

**目录结构**:
```
apps/frontend/src/pages/
└── adscenter/
    ├── index.tsx        → /adscenter（Ads中心主页面）
    └── callback.tsx     → /adscenter/callback（OAuth回调处理）
```

## ✅ 核心澄清总结

### 后端API路由 vs 前端页面路由

| 路由路径 | 类型 | 说明 |
|---------|------|------|
| `/api/v1/siterank` | ✅ 后端API | SiteRank评估服务，**无前端页面** |
| `/api/v1/batchopen` | ✅ 后端API | BatchOpen补点击服务，**无前端页面** |
| `/api/v1/adscenter` | ✅ 后端API | Google Ads API服务 |
| `/adscenter` | ✅ 前端页面 | Ads中心页面，调用 `/api/v1/adscenter` API |
| `/offers` | ✅ 前端页面 | Offer库页面，调用多个API（offers/siterank/batchopen） |
| `/dashboard` | ✅ 前端页面 | 大盘页面 |
| `/tasks` | ✅ 前端页面 | 任务中心页面 |

### 关键结论

1. ✅ **前端页面只有4个核心页面**: `/dashboard`, `/offers`, `/adscenter`, `/tasks`
2. ✅ **SiteRank和BatchOpen只有后端API**，没有独立前端页面
3. ✅ **所有功能都在对应页面通过API调用完成**，不需要跳转到其他页面
4. ✅ **OAuth回调路径**: `/adscenter/callback`（因为adscenter是独立前端页面）
